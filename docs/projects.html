Programming and utility tools

CPP_Utilities

C++20

A collection of C++ utility classes and functions, under the namespace "utils". It contains data structures, standard containers cout, coloured cout (on Windows only), mathematical utilities like a vec2 class, angles and transforms, an asynchronous logger and much more.
In my first attempt to make a 2D game engine, I started collecting some classes on the side, together with some other utilities I found out I kept reusing in my code in different projects. When I finally scrapped the game engine, I decided to keep some of the data structures written for it. The collection grew over time, to the state of today; I'm still actively adding funcitonality to it. I'm particularly proud of my tracking_ptr, a smart observer pointer which never loses its target, even when that target moves in memory.

Project

Plotter

C++17 / C# / C++CLI / SFML / LibreHardwareMonitorLib

Tired of monitoring software that is flat looking, lacks dark themes, and/or only shows plots in tiny little areas, I decided to make my own resource monitor. The looks are extremely customizable, but getting started is a bit slow. You first have to get a list of the hardware IDs of your machine that are supported by LibreHardwareMonitorLib (output on console when Plotter is launched). Then you need to create a script which defines your UI. "Plotter_data.txt" is the one working on my machine, and it's fully commented, giving all the information needed to modify it to fit your own machine. TODO Github readme.

Graph Maker

C++ / SFML

A program I made originally in Game Maker to have a simple visualization of algorithms during my algorithms and data structure course at university. As it usually happens with me, it exploded in features.
Now it is a Graph editor and visualizer, with file inport and export, plus AngelScript support that lets users run completely arbitrary algorithms written in AngelScript on the currently loaded graph. Properly written navigation algorithms will not only run on the graph, but also show in real time with color changes and lines animations how the graph is being visited.
Originally I planned writing an interpreter for an ad-hoc C-like language for the purpose, together with a course mate. We started it, but stopped when we arrived at implementing classes in our language. I've then decided to switch to AngelScript. Our old parser is in the "Craph" project mentioned later.

Image Viewer

C++17 / SFML

Since Windows 10's built in photo viewer takes ages to load, and all the image viewers I could find came with some unnecessary fluff, I said "fine, I'll do it myself". It doesn't do anything fancy at all. It just opens pictures, zooms in and out, scrolls to next and previous files, and lets you drag the image on the window. I'm planning making it a borderless draggable window, so that everything of the space it takes is dedicated to displaying the picture, no borders, no bars.
